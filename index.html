Usal Project

Designed by Chase Shewbridge

Intro

Usal Project is an experiences company based in Los Angeles, California. They began with the goal of organizing events for like minded people to meet, learn and experience new things. They offer in-person meet ups for all types of activities like Art shows, Yoga Classes, Backpacking, Fishing and way more.

The top half of this study covers the new version of the website. Below this study, there are some words and images regarding the first version of the site, created with WordPress.

next
prev

Brief

After utilizing the WordPress site we created together a year earlier, a lot had changed with Usal. The biggest factor being that it went from a side project to a full time gig for their owner. The fact that the website had become their number one source of income, there were things that we decided could be set up better. This prompted us to look at Shopify, for ease of use, better support and to make things easier for them to do on their own.

Functionality

One of the biggest hurdles was the Calendar and how to split up recurring events. Since there are multiple Class Instructors, Multiple locations, Multiple dates, we decided to connect the Class Instructor data to the Date and Location attributes, and have each Class Type be a single product. This way, our 'Escape' category doesn't look out of balance, for example with 8 of the same Yoga Class each month. There would instead be just one Yoga Class, with a variantion of Dates, each date with it's own Class Instructor and Location.

Technology

After digging through many different Appointment Booking Apps, Class Booking Apps, etc, I decided that Evey Events would be a solid option. They offered a way to sync their templates with my theme so I could basically reorganize the way the App was display it's data. With the Bookings->Products bridge set up, would run the site like a typical eCommerce website, and allow Class Instructors to log in and update their event information pretty easily. At the very least without doing any coding.

Version 01

Designed by Chase Shewbridge

The initial idea for this project would be for us to create a startup WordPress page, with some merchandise and three categories of products that would be repurposed as Events. There would need to be a full calendar view, and a more compact widget that goes on the home page. Products would be organized as Escape (trips), Learn (classes), and Goods (merchandise).

The main piece of the project was organizing their different events into categories, then creating a custom Taxonomy for each. With a custom Taxonomy, rather than custom product Categories, we would be able to utilize the built-in permalink structure that WordPress prefers to use. For example, if 'Hiking Trips' were a regular product category, we would need to rewrite the default shop page to not display Hiking Trips. In addition we'd need to rewrite the default shop categories pages to not display 'Hiking Trips', then write additional shop archive and category pages for those types of events. The result gave us a clean URL system, like /escape/our-trip-name, or /learn/our-inperson-class-name, rather than /shop/categories/escape/our-trip-name.

We decided to stick with WordPress on the first version of the site, because it seemed like this would be the more cost effective route, and frankly time effective because we were more experienced with WordPress at the time. So the bulk of the functionality comes from WooCommerce. The events utilize Google Calendar for all the scheduling details. The information is pulled from Google using a minimal iCal calendar plugin that maps the data to a calendar. The calendar was then reformated using CSS and JavaScript to suite the original design.

The main issues with the first website was the flow for adding new Events. We had initially set it up using Google Calendar to populate a JavaScript calendar, that linked to the individual Products that we re-purposed as Events with the use of custom Taxonomies. This worked but proved to be too many steps when adding multiple events, every day, for months. We decided to look into repurposing a Shopify App to handle a booking service.

All Is A Gentle Spring

Designed by William Canning

Intro

all is a gentle spring is an Australian label founded in 2018. The label warps, condenses and clarifies the vernacular of costume history; borrowing methods and materials from the theatre workshop and transposing them into the world of ready-to-wear.

Brief

Our job was to format a Shopify theme that was already partially created. Apparently, the first developer ran into some issues and they needed someone to come in a get a few things working. The first was the overall layout of the site, it didn’t match the design provided. In addition, we needed some hover effects, custom fonts and various alignment changes.

Functionality

Their business goals don’t stray far from what Shopify provided by default, so there was not much to do in the backend, besides categories the products to show up the way they wanted. I also added a metafield for the product image swap on hover for the Archive page.

Technology

This theme was built using Shopify and their CLI locally. The project is integrated with GitHub and has continous deployment set up.

Dominic Fike

Designed by Reed Bennett

Intro

Dominic Fike is a talented and world reknown singer, songwriter and actor. Their music has been featured in Fortnite, Euphoria and has been recognized with award nominations from The Grammys, and MTV.

Brief

The goal for this project was to get a splash page together that would serve as a portal for fans to find official Ticket links to each show in their area. The design was meant to replicate the flier created by Reed, with the addition of some Pixels and trackers provided by the Label and management teams.

Functionality

Since the site was already designed in a poster format, which is basically the opposite of a website, we decided it would be best to split it up into sections so we could allow the Browser to do all of it's responsiveness without skewing anything too much. The process was easy, as Reed had sent me the flier assets to work with. The only tricky thing was getting the resolution balanced enough to load quickly while not degrading too much. Since we wanted the site to help in regards to SEO, we didn't want to have everything be an image, so we found a Google font similar enough to the one they used on the flier.

Technology

The website did not need any CMS or anything, so I decided to use Netlify to host it. I had been using it to host my Portfolio and some other projects, with no issues. This is the first project with an issue haha. I was aware that Dominic had a large following, and that a lot of people would view this website. I was unaware as to how much optimization this would require in order to not overload the host. Within a couple of days it racked up a pretty hefty bill in bandwidth overages. The images, though not over 1500px wide were still not optimized enough to be called that many times without having an issue. After coordinating with the Netlify team we were able to optimize the images and remedy the charges thankfully. I believe every mistake is a chance to learn, and in that moment we learned the absolute importantance of image optimization.

The tour is over, but the website is still currently live, if you would like to take a look.

Bighouse

Based on the Brand Guidelines created by William Canning

Intro

Since I'm into synthesizers, I like to think of Bighouse as a semi-modular creative unit. While it's kind of a creative agency, they don't operate under one roof, or even under one set of goals. There is a leader but they don't dictate or manage the work. It's like a collective of creatives that help eachother out. We too have met great clients through the Bighouse label and strive contribute to it's growth when possible.

A few slides

Brief

The purpose of this project was to create a deck that can explain the company and it's goals in a much more coherent way than I did in the first paragraph. The different sectors of Bighouse would be layed out on their own slides, in an attempt to define each 'wing' of the company.

Technology

We agreed from the beginning to be in charge of future updates and handling of the pitch deck, so the technology used was our choice. We decided to use a combination of Illustrator and InDesign. Illustrator (as of recently Affinity Designer to be specific) is our preferred way of working with both vector graphics and also large PDF documents with multiple pages. InDesign was used toward the end of the project, to add Hyperlinks and page transitions to the PDF.

Thoughts

We've been told that the deck has been extremely effective both to get meetings, but also as a back bone to zoom calls and email coordination. Since the initial design, we have revised it twice and it continues to help further the goals of the company in a meaningful way.
